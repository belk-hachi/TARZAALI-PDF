# TARZALI-PDF

> ⚠️ **This is a private, single-lab internal tool. It is not designed for public use, multi-tenant deployment, or production-grade security. See the disclaimer below.**

TARZALI-PDF automates a specific medical laboratory workflow — from automatic email retrieval and AI-powered PDF parsing to individual patient report generation.

---

## 🧩 The Problem This Solves

Some independent labs in Algeria work under a **subcontracting agreement with Tarzaali**, a larger reference laboratory. When a patient needs a test the small lab can't run in-house, the sample is sent to Tarzaali, which processes it and sends back results.

Tarzaali delivers those results as a **batch PDF list** — one document containing all patients from that day, formatted in their own layout. The contracted lab needs to:

1. Extract each patient's results from that batch
2. Re-issue them as an **individual report that mirrors Tarzaali's own PDF design** — same layout, same look — as if it came directly from Tarzaali for that one patient
3. Print and hand it to the patient

**Before this tool**, a staff member would open the batch PDF, manually find each patient, and piece together an individual report by hand — a slow, error-prone process done daily.

**This tool eliminates that entirely.** The batch PDF arrives by email, gets picked up automatically, is parsed by AI into structured data, and individual reports are generated and ready to print — with no manual intervention.

The AI extraction isn't a gimmick: Tarzaali's batch PDF format is complex (multi-page patients, French medical terminology, annexe pages for specialist tests, abnormal value flags, pending/rejected statuses) and changes subtly over time. A rule-based parser would be brittle. Gemini reads it the way a human would.

**Who can actually use this** — if you're an Algerian lab contracted with Tarzaali and want to automate splitting their batch PDFs into individual patient reports that match their layout, this tool is directly applicable with minimal configuration.

---

## 🔬 Context & Scope

This tool is tightly coupled to one specific workflow and data format:

- It parses PDF reports generated by **the Tarzaali Online platform**.
- The AI extraction prompt (`prompt.md`) is hand-tuned to that platform's exact PDF layout and French-language field names.
- The external API integration points to a **hardcoded private endpoint** specific to Tarzaali Online.
- The email automation polls a **specific IMAP mailbox** for reports from a known sender.
- The generated PDF layout replicates Tarzaali's own report design.

If you're not working with Tarzaali's system, this tool will likely be useless out of the box — and that's by design.

## ⚠️ Security Disclaimer

This application was built for **local/intranet deployment on a single trusted Windows machine**, not for internet-facing or multi-user environments. Several deliberate simplifications were made because the threat model is "one lab, one PC, trusted local network":

| What | Why it's acceptable here | Why it would matter elsewhere |
|---|---|---|
| `SECRET_KEY` hardcoded as fallback | Single-machine, no session hijacking risk | Must be a strong random secret in production |
| Passwords Base64-obscured, not encrypted | Config file is on a trusted local disk | Real encryption (e.g. OS keychain) needed for shared/cloud systems |
| No authentication layer | Tool runs locally, only lab staff have access | Any internet exposure would need login/auth |
| SQLite, no connection pooling | Single concurrent user | Would need PostgreSQL + proper pooling for multi-user |
| `FLASK_DEBUG` via env var, no hardening | Dev/local use | A production Flask app needs a WSGI server (Gunicorn, etc.) |
| No input sanitization beyond path traversal | Trusted internal data sources | Untrusted input would require much stricter validation |

These are **not oversights** — they are conscious trade-offs for a tool that runs offline, serves one user, and handles non-public data in a controlled environment.

## 🚀 Key Features

- **Two Ways to Get the Batch PDF** — Either download the daily list manually from the Tarzaali website and upload it through the dashboard, or configure the email integration to have the app pick it up automatically from the lab's mailbox.
- **Background Email Automation** — Polls IMAP for lab reports, downloads PDF attachments, and merges them automatically.
- **AI-Powered Extraction** — Uses Google Gemini 2.5 Flash to parse French medical PDFs and extract structured patient/test data.
- **Smart PDF Generation** — Produces professional, branded patient reports via ReportLab, with automatic annexe (Ci-Joint) merging.
- **High-Density Dashboard** — Slate-themed Flask UI with real-time stats, search, filters, inline editing, and delivery tracking.
- **Maintenance & Reliability** — Compressed ZIP backups, activity logging, and background status polling.
- **Offline-Ready UI** — All Bootstrap assets are local; no internet required to run the interface.

## 🏁 Getting Started

```powershell
# 1. Clone
git clone <repo-url>
cd tarzali-pdf

# 2. Create virtual environment
python -m venv .venv
.venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Run
python app.py
```

Open `http://server-ip:5000` in your browser. Since the app binds to `0.0.0.0`, any machine on the local network can access it — useful when the app runs on a dedicated lab PC and staff access it from other workstations.

---

## ⚙️ Configuration

All settings are managed through the **⚙️ Paramètres** modal in the dashboard and saved to `config.json`:

- **API Key** — Gemini API key from [Google AI Studio](https://aistudio.google.com/)
- **Email** — IMAP server, credentials, sender/subject filters, fetch interval
- **Laboratory Info** — Doctor name, address, phone/fax for PDF headers
- **Backup** — Custom storage path (supports UNC network paths), max backup count
- **Prompt** — Edit `prompt.md` directly to tune AI extraction behavior

## 📦 Building a Standalone Executable

```powershell
pyinstaller --noconfirm --onefile --windowed `
    --add-data "logo.jpg;." `
    --add-data "templates;templates" `
    --add-data "static;static" `
    --name "TARZALI-PDF" `
    app.py
```

The output `.exe` in `dist/` is fully self-contained and runs without Python installed.

## 🏗️ Design Notes

The security simplifications listed above are intentional — not oversights. The deployment context (one machine, trusted local network, single user) doesn't justify the overhead of auth layers, a production WSGI server, or encrypted secret storage. A multi-user or internet-facing version of this would need those things, but that's not what this is.

AI extraction was chosen over rule-based PDF parsing because the reference lab's batch format is complex enough — multi-page patients, annexe pages, mixed statuses, subtle layout variations — that a regex or coordinate-based parser would require constant maintenance. Gemini reads it the way a human would, and the prompt in `prompt.md` can be updated without touching any code when the format changes.

The database layer is intentionally ORM-free. The queries are straightforward enough that SQLAlchemy would add abstraction without benefit, and raw `sqlite3` keeps the dependency footprint small for a PyInstaller-bundled `.exe`.
